# Docker Notes
    -docker is like a software/platform that builds your code in a way that it can fully run on any environment (shipping it).
    -in docker there is what we call a registry in which, like github, stores data but for docker images, which are essentially the blueprint of the container in which the code is shipped in. another thing about docker is a dockerfile, which lies below the image because it tells docker how to create an image. port mapping = connection to the shipped code, volume = localized storage (still not deep on this).
